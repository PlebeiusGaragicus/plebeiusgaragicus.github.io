How NIP-33 Works

Addressable Events are Nostr events that can be replaced by newer versions. They use:

  • Kind range: 30000-39999 (we're using 30078)
  • Required "d" tag: A unique identifier (we're using "current-status")
  • Replacement logic: Same pubkey + kind + "d" tag = newer event replaces older
  • Address format: <kind>:<pubkey>:<d-value>

Perfect for status updates, profiles, or any data that should be current rather than historical.
